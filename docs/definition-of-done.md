# Definition of Done

A task is done when:

1. Code is committed in the monorepo with clear structure and naming.
2. Local boot works with one command (`npm run local:boot`).
3. API health endpoint responds with HTTP 200 and status payload.
4. Seed script creates realistic demo data for all MVP entities.
5. Lint/format/test commands are available from the root workspace.
6. At least one Playwright smoke test scaffold exists and is runnable.
7. Product and architecture docs are updated.
8. Changes remain simple, readable, and aligned with MVP scope.
